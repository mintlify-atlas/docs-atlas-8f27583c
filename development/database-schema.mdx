## Overview

Lynx Note uses MongoDB as its database with Mongoid as the Object-Document Mapper (ODM). All models include `Mongoid::Timestamps` which automatically adds `created_at` and `updated_at` fields.

## Models

### User

The core model representing user accounts with authentication and relationships.

**Fields:**

| Field | Type | Description |
|-------|------|-------------|
| `name` | String | User's display name |
| `email` | String | User's email (must match email format) |
| `password` | String | BCrypt-hashed password |
| `role` | String | User role (e.g., 'admin', 'user') |
| `created_at` | DateTime | Auto-generated timestamp |
| `updated_at` | DateTime | Auto-generated timestamp |

**Relationships:**

**Fields:**

| Field | Type | Description |
|-------|------|-------------|
| `title` | String | Note title (optional if content exists) |
| `content` | Array | Array of JSON strings containing note blocks |
| `user_id` | BSON::ObjectId | Reference to owning user |
| `created_at` | DateTime | Auto-generated timestamp |
| `updated_at` | DateTime | Auto-generated timestamp |

**Relationships:**

```ruby
belongs_to :user
has_many :sharings, as: :shareable, dependent: :destroy
```

Source: `app/models/note.rb:7-9`

**Content Structure:**

The `content` field stores an array of JSON strings, where each string represents a content block:

```json
[
  "{\"type\": \"text\", \"value\": \"Note content here\"}",
  "{\"type\": \"image\", \"value\": \"image_url\"}"
]
```

**Validations:**
- Must have at least a title OR content with non-blank value
- Custom validation iterates through content array to check for non-empty values

### Sharing

Polymorphic model that handles sharing of Notes and Collections.

**Fields:**

| Field | Type | Description |
|-------|------|-------------|
| `shared_with` | Array | Array of user IDs who have access |
| `owner_id` | BSON::ObjectId | User who owns the shared item |
| `shareable_id` | BSON::ObjectId | ID of shared Note or Collection |
| `shareable_type` | String | Type of shared item ("Note" or "Collection") |
| `created_at` | DateTime | Auto-generated timestamp |
| `updated_at` | DateTime | Auto-generated timestamp |

**Relationships:**

```ruby
belongs_to :owner, class_name: 'User', inverse_of: :sharings
belongs_to :shareable, polymorphic: true  # Note or Collection
```

Source: `app/models/sharing.rb:5-8`

**Polymorphic Association:**

The `shareable` polymorphic association allows a single Sharing record to reference either a Note or Collection:

```ruby
# Sharing a Note
sharing = Sharing.new(
  owner: current_user,
  shareable: note,  # Note instance
  shared_with: [friend_id]
)

# Sharing a Collection
sharing = Sharing.new(
  owner: current_user,
  shareable: collection,  # Collection instance
  shared_with: [friend_id]
)
```

## Data Integrity

### Cascade Deletion

All relationships use `dependent: :destroy` to maintain referential integrity:

- Deleting a **User** destroys:
  - All their notes
  - All their collections
  - All their notifications
  - All initiated and received friendships
  - All sharings they own

- Deleting a **Note** or **Collection** destroys:
  - All associated sharing records

- Deleting a **Friendship** triggers:
  - Cleanup of sharing records (removes friend from shared_with arrays)

### Custom Cleanup

The User model implements custom cleanup before deletion:

```ruby
before_destroy :remove_friendships

def remove_friendships
  self.initiated_friendships.destroy_all
  Friendship.where(friend_id: self.id).destroy_all
end
```
