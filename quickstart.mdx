## Overview

This guide will walk you through the essential features of Lynx Note:

1. Creating your account
2. Creating your first note
3. Organizing notes in a collection
4. Adding friends
5. Sharing content with friends

By the end, you'll be ready to use Lynx Note for your collaborative note-taking needs.

## Step 1: Create Your Account

<Steps>
  <Step title="Navigate to Sign Up">
    Open your browser and go to:
    
    ```
    http://localhost:3000/signup
    ```
    
    Or click the "Sign Up" link from the home page.
  </Step>
  
  <Step title="Fill in Your Details">
    The registration form requires three fields:
    
    - **Name**: Your display name (required)
    - **Email**: A valid email address (required, validated)
    - **Password**: Your secure password (required, will be encrypted)
    
    <CodeGroup>
    ```ruby User Model Validation
    # From app/models/user.rb
    field :name, type: String
    field :email, type: String
    field :password, type: String
    field :role, type: String  # Automatically set to 'user'
    
    # Email must match valid format
    if email.present? && !email.match?(URI::MailTo::EMAIL_REGEXP)
      errors.add(:email, "It is invalid, does not have the format")
    end
    ```
    </CodeGroup>
  </Step>
  
  <Step title="Submit Registration">
    Click the registration button. If successful, you'll see:
    
    ```
    "Registered, you can now log in"
    ```
    
    <Note>
    Your password is automatically encrypted using BCrypt before being stored in the database. The system uses BCrypt's secure hashing with salt.
    </Note>
  </Step>
</Steps>

### Login to Your Account

After registration, log in at `http://localhost:3000/login`:

```ruby Session Authentication
# From app/controllers/session_controller.rb
# The system authenticates your email and password
user = User.find_by(email: params[:email])
if user && user.authenticate(params[:password])
  session[:user_id] = user.id
  # Redirect to notes dashboard
end
```

## Step 2: Create Your First Note

<Steps>
  <Step title="Access the Notes Section">
    After logging in, navigate to:
    
    ```
    http://localhost:3000/notes
    ```
    
    Click the "New Note" button.
  </Step>
  
  <Step title="Add a Title">
    Enter a descriptive title for your note:
    
    ```
    My First Note
    ```
    
    <Note>
    You must provide either a title or content. A completely empty note cannot be saved.
    </Note>
  </Step>
  
  <Step title="Add Content">
    Lynx Note supports three types of content blocks:
    
    <Tabs>
      <Tab title="Text">
        Add text paragraphs to your note:
        
        ```json Content Format
        {
          "type": "text",
          "value": "This is my first note in Lynx Note!"
        }
        ```
      </Tab>
      
      <Tab title="List">
        Create lists (items are semicolon-separated):
        
        ```json Content Format
        {
          "type": "list",
          "value": "Task 1;Task 2;Task 3"
        }
        ```
        
        The UI will display these as separate list items.
      </Tab>
      
      <Tab title="Image">
        Upload images to your note:
        
        ```json Content Format
        {
          "type": "file",
          "value": "/uploads/abc123-def456.jpg"
        }
        ```
        
        Images are stored with unique UUIDs in `public/uploads/`.
      </Tab>
    </Tabs>
  </Step>
  
  <Step title="Save Your Note">
    Click "Create Note" to save. The note is stored in MongoDB:
    
    ```ruby Note Creation
    # From app/controllers/notes_controller.rb
    @note = Note.new(title: note_params[:title])
    @note.user = current_user
    
    # Process content blocks
    processed_content = params[:note][:content].map do |content_item|
      case content_item[:type]
      when 'text'
        content_item if !content_item[:value].blank?
      when 'list'
        { type: 'list', value: content_item[:value].reject(&:blank?).join(';') }
      when 'file'
        { type: 'file', value: handle_uploaded_file(content_item[:value]) }
      end
    end.compact
    
    @note.content = processed_content.map { |item| item.to_json }
    @note.save
    ```
  </Step>
</Steps>

### Edit Your Note

To edit an existing note:

1. Click on the note from your notes list
2. Click the "Edit" button
3. Modify the title or content
4. Update or delete existing images
5. Click "Update Note"

<Warning>
When you delete an image from a note, the file is permanently removed from the server. This action cannot be undone.
</Warning>

## Step 3: Organize Notes in Collections

Collections help you organize related notes together.

<Steps>
  <Step title="Create a Collection">
    Navigate to:
    
    ```
    http://localhost:3000/collections
    ```
    
    Click "New Collection" and enter a name:
    
    ```
    Work Projects
    ```
    
    ```ruby Collection Model
    # From app/models/collection.rb
    class Collection
      field :name, type: String
      field :notes, type: Array, default: []
      
      belongs_to :user
      has_many :sharings, as: :shareable
    end
    ```
  </Step>
  
  <Step title="Add Notes to Collection">
    When creating or editing a note, you can select which collections it belongs to:
    
    1. Create or edit a note
    2. Check the boxes for the collections you want
    3. Save the note
    
    ```ruby Adding to Collections
    # From app/services/collections_notes_service.rb
    # The service handles adding notes to multiple collections
    service = CollectionsNotesService.new(@note)
    service.add_collections(note_params[:collection_ids])
    ```
  </Step>
  
  <Step title="View Collection Contents">
    Click on a collection to see all notes within it. Collections automatically update when you:
    
    - Add new notes
    - Remove notes
    - Delete notes entirely
  </Step>
</Steps>

## Step 4: Add Friends

Before you can share notes, you need to establish friendships.

<Steps>
  <Step title="Send a Friend Request">
    Navigate to:
    
    ```
    http://localhost:3000/friendships
    ```
    
    Click "New Friendship" and select a user from the list.
    
    ```ruby Friendship Creation
    # From app/controllers/friendships_controller.rb
    @friend = User.find(params[:friend])
    @friendship = current_user.initiated_friendships.build(friend: @friend)
    
    if @friendship.save
      # Create notification for recipient
      Notification.create(
        user: @friend,
        message: "#{current_user.name.capitalize} sent you a friend request.",
        notification_type: 'friend_request'
      )
    end
    ```
    
    The request starts with `status: "pending"`.
  </Step>
  
  <Step title="Accept or Decline Requests">
    If someone sends you a friend request, you'll see it in your friendships page:
    
    - **Accept**: Changes status to "accepted" and allows sharing
    - **Decline**: Deletes the friendship request
    
    ```ruby Accept Request
    # From app/controllers/friendships_controller.rb
    @friendship = Friendship.find(params[:id])
    if @friendship.status == 'pending'
      @friendship.update(status: 'accepted')
      
      # Notify the requester
      Notification.create(
        user: @friendship.user,
        message: "#{current_user.name.capitalize} has accepted your friendship request.",
        notification_type: 'friend_request'
      )
    end
    ```
  </Step>
  
  <Step title="View Your Friends">
    Once accepted, friends appear in your friendships list. You can see:
    
    - Friend's name
    - Notes they've shared with you
    - Collections they've shared with you
  </Step>
</Steps>

### Friendship Relationships

Lynx Note tracks friendships bidirectionally:

```ruby Friendship Model
# From app/models/user.rb
has_many :initiated_friendships, 
  class_name: 'Friendship', 
  inverse_of: :user
  
has_many :received_friendships, 
  class_name: 'Friendship', 
  inverse_of: :friend
```

## Step 5: Share Notes and Collections

Now that you have friends, you can share content with them.

<Steps>
  <Step title="Share a Note">
    From the note view, click the "Share" button:
    
    1. Select one or more friends to share with
    2. Click "Share Note"
    3. Your friends receive a notification
    
    ```ruby Sharing Creation
    # From app/controllers/sharing_controller.rb
    @sharing = Sharing.new
    @sharing.owner = current_user
    @sharing.shareable = @note  # Polymorphic association
    @sharing.shared_with = [friend1_id, friend2_id]
    
    if @sharing.save
      # Notify each friend
      @sharing.shared_with.each do |friend_id|
        user = User.find(friend_id)
        Notification.create(
          user: user.id,
          message: "#{current_user.name.capitalize} has shared a note with you",
          notification_type: 'note_request'
        )
      end
    end
    ```
  </Step>
  
  <Step title="Share a Collection">
    Sharing collections works the same way:
    
    1. View a collection
    2. Click "Share Collection"
    3. Select friends
    4. Click "Share"
    
    When you share a collection, friends can see all notes within it.
  </Step>
  
  <Step title="Edit Sharing Permissions">
    You can modify who has access at any time:
    
    1. Click "Edit Sharing" on a shared item
    2. Add or remove friends from the list
    3. Save changes
    
    <Note>
    When you remove someone from sharing, they receive a notification and immediately lose access to the content.
    </Note>
  </Step>
  
  <Step title="Unshare Completely">
    To stop sharing with everyone:
    
    1. Click "Delete Sharing"
    2. All friends lose access
    3. Notifications are sent to inform them
    
    ```ruby Unsharing
    # From app/controllers/sharing_controller.rb
    @sharing = Sharing.find(params[:id])
    @sharing.destroy
    
    # Notify all friends who had access
    @sharing.shared_with.each do |friend_id|
      user = User.find(friend_id)
      Notification.create(
        user: user.id,
        message: "#{current_user.name.capitalize} has deleted a note with you",
        notification_type: 'note_request'
      )
    end
    ```
  </Step>
</Steps>

### Understanding the Sharing Model

Sharing is implemented using polymorphic associations:

```ruby Sharing Model
# From app/models/sharing.rb
class Sharing
  belongs_to :owner, class_name: 'User'
  belongs_to :shareable, polymorphic: true  # Can be Note or Collection
  
  field :shared_with, type: Array, default: []
end
```

This allows the same sharing mechanism to work for both notes and collections.

## View Shared Content

To see what others have shared with you:

<Steps>
  <Step title="Check Your Friendships Page">
    Navigate to:
    
    ```
    http://localhost:3000/friendships
    ```
    
    Each friend shows:
    - Notes they've shared with you
    - Collections they've shared with you
  </Step>
  
  <Step title="Access Shared Notes">
    Click on any shared note to view and edit it:
    
    ```ruby Shared Access Check
    # From app/controllers/notes_controller.rb
    def is_shared
      @note = Note.find(params[:id])
      sharing = Sharing.where(shareable_id: @note.id).first
      
      # Check if current user has access
      unless @note.user_id == current_user.id || 
             sharing&.shared_with&.include?(current_user.id.to_s)
        redirect_to root_url
      end
    end
    ```
  </Step>
  
  <Step title="View Shared Collections">
    Shared collections appear in your collections list:
    
    ```ruby Shared Collections
    # From app/models/user.rb
    def shared_collections
      collection_ids = Sharing.where(
        :shared_with.in => [self.id.to_s],
        shareable_type: 'Collection'
      ).pluck(:shareable_id)
      
      Collection.find(collection_ids)
    end
    ```
  </Step>
</Steps>

## Mobile Access

Lynx Note is fully responsive and works on mobile devices:

```ruby Browser Detection
# From app/controllers/notes_controller.rb
@browser = Browser.new(request.user_agent)

if browser.device.mobile?
  # Mobile-optimized redirects and views
  redirect_to @note, notice: 'Note was successfully created.'
else
  # Desktop view with sidebar
  redirect_to notes_path(number: @note.id)
end
```
