## Overview

Lynx Note's sharing system enables you to share notes and collections with your friends. The sharing model uses polymorphic associations to handle both notes and collections uniformly.

## Sharing Model

### Data Structure

The sharing system is built on a polymorphic association:

```ruby app/models/sharing.rb
class Sharing
  include Mongoid::Document
  include Mongoid::Timestamps

  belongs_to :owner, class_name: 'User', inverse_of: :sharings
  belongs_to :shareable, polymorphic: true

  field :shared_with, type: Array, default: []
end
```

### Key Components

<CardGroup cols={3}>

<Info>

## Prerequisites

Before you can share content, you must:

1. Have at least one accepted friendship (see [Friendships](/features/friendships))
2. Own the note or collection you want to share

### Notification on Update

Friends receive notifications when their access changes:

- **Added friends**: Notified about new shared content
- **Removed friends**: Notified about removed access

```ruby app/controllers/sharing_controller.rb:111
removed_friend_ids.each do |id|
  messageContent = "The user #{current_user.name.capitalize} has deleted a note with you"
  Notification.create(user: user.id, message: messageContent, notification_type: type)
end
```

<Warning>
  You cannot remove all friends from a share via update. To completely stop sharing, you must delete the sharing record.
</Warning>

## Unsharing (Recipient Action)

Friends who receive shared content can remove themselves from the share:

```ruby app/controllers/sharing_controller.rb:179
def unsharing
  @sharing = Sharing.find(params[:id])
  @sharing.pull(shared_with: current_user.id.to_s)
  @sharing.save
end
```

This action:
- Removes the friend's ID from the `shared_with` array
- Does NOT notify the owner
- Immediately revokes access to the content

<Tip>
  Use the unsharing feature to decline access to shared content you're no longer interested in.
</Tip>

## Access Control

### Viewing Shared Content

When accessing a shared note or collection, the system verifies:

```ruby app/controllers/notes_controller.rb:14
sharing = Sharing.where(shareable_id: @note.id).first
@friend_sharing = @note.sharings.any? && sharing&.shared_with&.include?(current_user.id.to_s)

unless @note.user_id == current_user.id || sharing&.shared_with&.include?(current_user.id.to_s)
  redirect_to root_url
end
```

Access is granted if:
- You are the owner, OR
- Your user ID is in the sharing's `shared_with` array

Friends with access can edit shared notes and collections:

## Friendship Integration

The sharing system is tightly integrated with friendships:

### Cleanup on Friendship Deletion

When a friendship is destroyed, all related sharings are cleaned up:

```ruby app/models/friendship.rb:16
def cleanup_sharings
  Sharing.where(:shared_with.in => [self.friend.id.to_s]).each do |sharing|
    sharing.pull(shared_with: self.friend.id.to_s)
  end
  Sharing.where(:shared_with.in => [self.user.id.to_s]).each do |sharing|
    sharing.pull(shared_with: self.user.id.to_s)
  end
end
```

This ensures:
- Former friends lose access to your shared content
- You lose access to content they shared with you
- No orphaned sharing relationships remain

From `app/models/user.rb:61-74`:
```ruby
def shared_notes_by(friend)
  note_ids = Sharing.where(owner: friend, shared_with: self.id.to_s, shareable_type: 'Note').pluck(:shareable_id)
  Note.where(:id.in => note_ids)
end

def shared_collections_by(friend)
  collection_ids = Sharing.where(owner: friend, shared_with: self.id.to_s, shareable_type: 'Collection').pluck(:shareable_id)
  Collection.where(:id.in => collection_ids)
end
```
