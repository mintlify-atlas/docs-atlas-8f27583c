## Overview

Lynx Note includes a production-ready Dockerfile that packages the application with all dependencies. This guide covers deploying with Docker and Docker Compose.

## Dockerfile Architecture

The Dockerfile uses a multi-stage build process to minimize the final image size:

### Build Stages

1. **Base Stage**: Sets up Ruby 3.0.2 environment
2. **Build Stage**: Compiles gems and assets
3. **Final Stage**: Creates minimal production image

## Prerequisites

- Docker Engine 20.10 or higher
- Docker Compose 2.0 or higher (optional)
- MongoDB instance (separate container or external)

## Quick Start

<Steps>

<Step title="Clone the repository">

```bash
git clone <repository-url>
cd lynx-note
```

</Step>

<Step title="Build the Docker image">

```bash
docker build -t lynx-note:latest .
```

The build process:
- Installs Ruby 3.0.2
- Installs system dependencies (build-essential, git, libvips)
- Bundles gems
- Precompiles Bootsnap cache
- Precompiles Rails assets

</Step>

<Step title="Run with Docker">

```bash
docker run -d \
  -p 3000:3000 \
  -e MONGODB_URI=mongodb://mongo:27017/laboratorio_production \
  -e RAILS_ENV=production \
  -e SECRET_KEY_BASE=your_secret_key_here \
  --name lynx-note \
  lynx-note:latest
```

</Step>

<Step title="Verify deployment">

```bash
# Check container logs
docker logs lynx-note

# Access the application
curl http://localhost:3000
```

</Step>

</Steps>

## Docker Compose Setup

For a complete deployment with MongoDB:

<CodeGroup>

```yaml docker-compose.yml
version: '3.8'

services:
  mongodb:
    image: mongo:6.0
    container_name: lynx-note-mongodb
    restart: unless-stopped
    environment:
      MONGO_INITDB_DATABASE: laboratorio_production
    volumes:
      - mongodb_data:/data/db
    ports:
      - "27017:27017"
    networks:
      - lynx-note-network

  app:
    build: .
    container_name: lynx-note-app
    restart: unless-stopped
    ports:
      - "3000:3000"
    environment:
      RAILS_ENV: production
      MONGODB_URI: mongodb://mongodb:27017/laboratorio_production
      SECRET_KEY_BASE: ${SECRET_KEY_BASE}
      RAILS_LOG_LEVEL: info
    depends_on:
      - mongodb
    networks:
      - lynx-note-network
    volumes:
      - ./public/uploads:/rails/public/uploads

volumes:
  mongodb_data:

networks:
  lynx-note-network:
    driver: bridge
```

```env .env.example
SECRET_KEY_BASE=your_production_secret_key_base_here
RAILS_LOG_LEVEL=info
RAILS_MAX_THREADS=5
WEB_CONCURRENCY=2
```

</CodeGroup>

### Running with Docker Compose

<Steps>

<Step title="Create environment file">

</Step>

The application will be available at `http://localhost:3000`

## Dockerfile Breakdown

<CodeGroup>

```dockerfile Dockerfile
# Base image with Ruby 3.0.2
ARG RUBY_VERSION=3.0.2
FROM registry.docker.com/library/ruby:$RUBY_VERSION-slim as base

WORKDIR /rails

# Production environment variables
ENV RAILS_ENV="production" \
    BUNDLE_DEPLOYMENT="1" \
    BUNDLE_PATH="/usr/local/bundle" \
    BUNDLE_WITHOUT="development"
```

```dockerfile Build Stage
FROM base as build

# Install build dependencies
RUN apt-get update -qq && \
    apt-get install --no-install-recommends -y \
    build-essential git libvips pkg-config

# Install gems
COPY Gemfile Gemfile.lock ./
RUN bundle install && \
    rm -rf ~/.bundle/ "${BUNDLE_PATH}"/ruby/*/cache

# Copy application and precompile
COPY . .
RUN bundle exec bootsnap precompile app/ lib/
RUN SECRET_KEY_BASE_DUMMY=1 ./bin/rails assets:precompile
```

```dockerfile Runtime Stage
FROM base

# Runtime dependencies only
RUN apt-get update -qq && \
    apt-get install --no-install-recommends -y \
    curl libsqlite3-0 libvips && \
    rm -rf /var/lib/apt/lists /var/cache/apt/archives

# Copy built artifacts
COPY --from=build /usr/local/bundle /usr/local/bundle
COPY --from=build /rails /rails

# Run as non-root user
RUN useradd rails --create-home --shell /bin/bash && \
    chown -R rails:rails db log storage tmp
USER rails:rails

EXPOSE 3000
ENTRYPOINT ["/rails/bin/docker-entrypoint"]
CMD ["./bin/rails", "server"]
```

</CodeGroup>

## Docker Entrypoint Script

The entrypoint script (`/home/daytona/workspace/source/bin/docker-entrypoint:1`) automatically prepares the database:

```bash
#!/bin/bash -e

# If running the rails server then create or migrate existing database
if [ "${1}" == "./bin/rails" ] && [ "${2}" == "server" ]; then
  ./bin/rails db:prepare
fi

exec "${@}"
```

## Environment Variables

### Required

| Variable | Description | Example |
|----------|-------------|----------|
| `MONGODB_URI` | MongoDB connection string | `mongodb://mongo:27017/laboratorio_production` |
| `SECRET_KEY_BASE` | Rails secret for encryption | Generated via `rails secret` |
| `RAILS_ENV` | Rails environment | `production` |

### Optional

| Variable | Description | Default |
|----------|-------------|----------|
| `PORT` | Application port | `3000` |
| `RAILS_LOG_LEVEL` | Log verbosity | `info` |
| `RAILS_MAX_THREADS` | Puma thread count | `5` |
| `WEB_CONCURRENCY` | Puma worker count | Auto (CPU cores) |

## Production Best Practices

### Security

- Application runs as non-root `rails` user
- Build dependencies removed from final image
- SSL enforced in production (requires reverse proxy)

### Performance

- Multi-stage build reduces image size
- Bootsnap precompilation for faster boot
- Assets precompiled during build
- Worker count auto-adjusts to CPU cores

### Persistence

Mount volumes for persistent data:

```yaml
volumes:
  # MongoDB data
  - mongodb_data:/data/db
  # Uploaded files (note images)
  - ./public/uploads:/rails/public/uploads
  # Logs
  - ./log:/rails/log
```

## Troubleshooting

### Container won't start

```bash
# Check logs
docker logs lynx-note

# Common issues:
# - Missing SECRET_KEY_BASE
# - MongoDB connection failed
# - Port 3000 already in use
```

### MongoDB connection issues

```bash
# Test MongoDB connectivity from app container
docker exec lynx-note curl mongodb:27017

# Verify MongoDB is running
docker ps | grep mongo
```

### Asset compilation failures

```bash
# Rebuild without cache
docker build --no-cache -t lynx-note:latest .
```

## Scaling

### Horizontal Scaling

```yaml
services:
  app:
    deploy:
      replicas: 3
    # ... rest of configuration
```

### Load Balancing

Add Nginx reverse proxy:

```yaml
  nginx:
    image: nginx:alpine
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./nginx.conf:/etc/nginx/nginx.conf:ro
    depends_on:
      - app
```
