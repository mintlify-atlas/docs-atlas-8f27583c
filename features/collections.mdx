## Overview

Collections help you organize notes into logical groups. A collection acts as a container that can hold multiple notes, and notes can belong to multiple collections simultaneously.

## Collection Structure

### Data Model

Collections use MongoDB with dynamic attributes:

```ruby app/models/collection.rb
class Collection
  include Mongoid::Document
  include Mongoid::Timestamps
  include Mongoid::Attributes::Dynamic

  field :name, type: String
  field :notes, type: Array, default: []
  has_many :sharings, as: :shareable, dependent: :destroy

  belongs_to :user
end
```

<Info>
  The `notes` field stores an array of note IDs (BSON::ObjectId), creating a many-to-many relationship without a traditional join table.
</Info>

### Validation

Collections must have a name (from `app/models/collection.rb:20`):

```ruby
def validate_title
  if name.blank?
    errors.add(:base, "You must provide a title.")

end
```

<Warning>

</Warning>

## Managing Notes in Collections

### Adding Notes

Notes are added to collections through the note creation/editing interface:

1. When creating or editing a note, you can select collections
2. The `CollectionsNotesService` handles the relationship:

```ruby
service = CollectionsNotesService.new(@note)
service.add_collections(collection_ids)
```

3. The note ID is pushed to the collection's `notes` array:

```ruby app/services/collections_notes_service.rb:18
collection.notes.push(@note.id)
collection.save
```

### Removing Notes

Notes are removed from collections in several scenarios:

<Tabs>
  <Tab title="Manual Removal">
    When editing a note, deselect a collection to remove the note from it:
    ```ruby
    service.update_collections(desired_collection_ids)
    ```
  </Tab>
  <Tab title="Note Deletion">
    When a note is deleted, it's automatically removed from all collections:
    ```ruby
    service.delete_collections
    ```
  </Tab>
</Tabs>

## Viewing Collections

### List View

Access your collections at `/collections` to see:
- Your own collections
- Collections shared with you
- Number of notes in each collection (sorted by size)
- Sharing status

### Collection Data Structure

The index view enhances each collection with runtime attributes:

```ruby app/controllers/collections_controller.rb:32
collection.assign_attributes(notes: notes)
collection.assign_attributes(sharing: {
  shared: collection.sharings.any?,
  id: collection.sharings.first&.id,
  is_shared: !current_user.collections.include?(collection)
})
```

<Note>
  Collections are sorted by the number of notes they contain (descending) for easy access to your most-used collections.
</Note>

### Detail View

When viewing a shared collection at `/collections/:id`:
- Only accessible if the collection is shared with you
- Shows all notes within the collection
- Provides the sharing ID for management

## Access Control

### Ownership Verification

Only collection owners can delete collections (from `app/controllers/collections_controller.rb:7`):

```ruby
def is_mine
  @collection = Collection.find(params[:id])
  unless @collection.user_id == current_user.id
    redirect_to root_url
  end
end
```

### Shared Access

You can view and edit a collection if (from `app/controllers/collections_controller.rb:13`):
- You own the collection, OR
- The collection has been shared with you

```ruby
sharing = Sharing.where(shareable_id: @collection.id).first
@friend_sharing = sharing&.shared_with&.include?(current_user.id.to_s)
```

## Deleting Collections

<Steps>
  <Step title="Verify Ownership">
    Only the collection owner can delete it
  </Step>
  <Step title="Remove Sharings">
    All sharing relationships are automatically destroyed (cascade delete)
  </Step>
  <Step title="Delete Collection">
    The collection document is removed from MongoDB
  </Step>
</Steps>

<Warning>
  Deleting a collection does NOT delete the notes within it. Notes remain intact and accessible.
</Warning>

## Sharing Collections

Collections support the polymorphic sharing system:

```ruby app/models/collection.rb:8
has_many :sharings, as: :shareable, dependent: :destroy
```

This allows collections to be shared with friends just like individual notes. See the [Sharing documentation](/features/sharing) for details.

## Collections & Notes Service

The `CollectionsNotesService` provides centralized logic for managing note-collection relationships:

### Service Methods

<CodeGroup>
```ruby Get Collections
# Find which collections contain a note
service.get_collections(collections)
# Returns array of collection IDs
```

```ruby Add to Collections
# Add note to specified collections
service.add_collections(collection_ids)
```

```ruby Update Collections
# Sync note's collection memberships
service.update_collections(desired_collection_ids)
# Removes from unselected, adds to selected
```

```ruby Remove from All
# Remove note from all collections
service.delete_collections
```
</CodeGroup>
